# Airport Management System (Lab #3)

## Обзор проекта
Это продвинутая система управления авиапарком аэропорта, написанная на Java. Проект демонстрирует переход от простого наследования к гибкой архитектуре на основе интерфейсов, а также реализацию полноценного жизненного цикла данных (хранение, экспорт, обработка).

## Ключевой функционал
* **Уровни доступа:** Система авторизации с разделением на `ADMIN` (полный доступ: редактирование, удаление, экспорт) и `USER` (только чтение и поиск).
* **Умное управление парком:** * Добавление разных типов техники (Самолеты, Вертолеты, Дроны).
    * Сортировка по модели и дальности полета.
    * Поиск и фильтрация через **Java Stream API**.
    * Статистика по типам аппаратов и ТОП-3 по характеристикам.
* **Интерфейсы поведения:** Реализация интерфейсов `Flyable` и `Maintainable`, позволяющая динамически определять возможности техники (например, дрон умеет летать, но не требует сложного ТО).
* **Работа с данными:**
    * **Сериализация:** Сохранение всего состояния системы в бинарный файл (`.dat`).
    * **Экспорт:** Выгрузка базы данных в формат `CSV` для работы в Excel.

## Технические особенности (Architecture)
Данная работа построена на строгом соблюдении принципов ООП:
1.  **Абстракция:** Базовый класс `Aircraft` задает шаблон, который нельзя инстанцировать.
2.  **Полиморфизм:** Использование `instanceof` и приведения типов для вызова специфичного поведения интерфейсов.
3.  **Инкапсуляция:** Все поля классов скрыты (`private`), доступ организован через геттеры и сеттеры.
4.  **Logging:** Внедрена библиотека **Apache Log4j** для детального протоколирования действий и ошибок (стек-трейсы не «валятся» в консоль, а пишутся в лог).
5.  **Robust Input:** Собственные методы `readInt` и `readDouble` гарантируют защиту от падения программы при некорректном вводе пользователя.

## Структура проекта
* `Main.java` — Консольный интерфейс и обработка команд.
* `AirportManager.java` — Бизнес-логика, управление коллекцией и работа с файлами.
* `Aircraft.java` — Иерархия моделей техники.
* `Flyable.java` / `Maintainable.java` — Интерфейсы поведения.
* `AuthService.java` — Логика регистрации и входа.

## Запуск
1.  Склонируйте репозиторий.
2.  Добавьте `log4j-1.2.17.jar` в библиотеки проекта.
3.  Убедитесь, что файл `log4j.properties` находится в корневой папке или в `src/resources`.
4.  Запустите класс `Main`.
